## REMOVED Requirements

### Requirement: The Reconcile Loop Composes The Family Over Public APIs
**Reason**: The existing requirement is expressed in workspace steps and code outcomes.
**Migration**: The deliberation loop composes only family mechanics honestly exercised by durable NLP invocations and dossier residuals.

### Requirement: Convergence Is Decided By Suunta
**Reason**: The existing convergence target is a verified code goal.
**Migration**: If retained, suunta evaluates unresolved questions, dissent, risks, and conditions; readiness never grants approval.

### Requirement: Each Step Executes Exactly Once
**Reason**: Code steps are removed.
**Migration**: Exactly-once applies to an identified Agent CLI invocation against one fixed input revision.

### Requirement: A Failed Step Retries Via Deferred Reclaim
**Reason**: Code-step retry semantics are removed.
**Migration**: Infrastructure-failed invocations may be reclaimed under locked limits without duplicating a witnessed answer.

### Requirement: The Composition Is Self-Checked
**Reason**: The current self-check asserts the removed code reconcile shape.
**Migration**: Self-check the retained family composition against synchronous dossier turns and recovery.

### Requirement: Step Execution Is Delegated To A Runner Seam
**Reason**: Ringi no longer owns a Step runner.
**Migration**: All semantic work is delegated to opaque respondent, arbitration, and evaluator AgentAdapter roles.

### Requirement: The Loop Reconciles A Changing Bearing Of Goal Plus Findings
**Reason**: Code-review findings and a verified code goal are removed.
**Migration**: The dossier residual consists of unresolved questions, dissent, risks, and fixed conditions.

### Requirement: Two Certifiers Feed The Sounding
**Reason**: Reviewer and command-verification certifiers are removed.
**Migration**: Independent arbitration updates public residuals; isolated evaluators judge fixed conditions; humans decide.

### Requirement: The Exactly-Once Attempt Identity Is Distinct From The Target Identity
**Reason**: The existing identity is expressed in code rounds and targets.
**Migration**: Invocation identity binds dossier, role, input revision or condition, turn, and attempt independently from the semantic issue identity.

### Requirement: The Round Loop Runs End-To-End Under Agent-Backed Roles
**Reason**: Builder/Reviewer/Verify rounds are removed.
**Migration**: Exercise an end-to-end synchronous dossier from submitted proposal through archived human decision.
